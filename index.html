Mybatis代码生成器无法生成xxxByPrimaryKey

教训

以后在数据库建表的时候，给表取的名字一定是要和该数据库相关联的，并且最好是在该数据库中不存在同名的表的

否则使用 Mybatis 代码生成器的时候就会出现一些莫名其妙的问题

踩坑经过

之前生成 ebook 表相关类的时候，xxxByPrimaryKey 都还在，不知道这次生成 category 表的时候却都没了

查看生成日志发现了可能的问题

这里提示说在 sakila 这个库中含有同样名为 category 的表

确实是这样的，于是我就想，是不是因为这个原因导致的代码生成器没办法生成 xxxByPrimaryKey 的方法了呢

于是我就将该数据据中的原 category 表更名为 wiki_category 后，再用代码生成器生成代码之后，就没报错了

xxxByPrimaryKey 方法也都回来了

之前为什么是正常的，应该是因为在所有数据库中还没有和 ebook 这张表同名的表

Vim编辑器操作

发表于 2023-02-09 更新于 2023-02-13 分类于 Linux

在 vim 编辑器中搜索关键词 “keyword”

  • 在搜索的结果中跳到下一个：N
  • 在搜索的结果中跳到前一个：n

在 vim 编辑器中

  • 显示行号：:set nu

  • 跳到行号为 n 的行：nG

  • 跳到文件的底部：G

  • 跳到文件的头部：g

  • 删除当前行在内的往下 n 行：ndd

  • 复制从光标开始往右的 n 个单词：nyw

  • 将包括当前行在内的往下 n 行复制到缓冲区中：nyy

  • 撤销上一步操作：u

  • 粘贴缓冲区（剪切板）里的内容到光标后：p

发表于 2023-02-09 更新于 2023-02-13 分类于 nodejs

NVM 是什么？

NVM 全称为 node.js version management，即 node.js 的版本管理

NVM 可以干什么？

通过 NVM 可以 安装和切换不同版本的 node.js

NVM 是用来解决什么问题的？

如果常用的 node.js 版本与项目中使用的的版本不兼容的话，就会报错，导致项目无法运行或其他的一些问题

如果这时候又想保留常用的版本，但又要暂时对这个项目进行开发的话，就会很头痛

NVM 就可以解决这个问题

我们可以通过 NVM 下载并安装多个版本的 node.js 到本地，并且能在这些版本中进行切换

这样，就解决了我们在上面遇到的问题
